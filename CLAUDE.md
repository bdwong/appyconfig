# Development Guidelines

## Methodology

- **TDD (Test-Driven Development)**: Write tests first, red-green, then refactor. Code coverage target is 90%.
- **OOP and Design Patterns**: Use object-oriented programming and established design patterns for abstraction.

## Documentation

- **README.md** documents usage and must be kept up to date. Structure: start with simple examples to get a user going, then introduce additional features to enable more complex workflows.
